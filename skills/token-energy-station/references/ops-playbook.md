# Ops Playbook

## 默认商品层级

### 高质量版

- 路由名：`premium-coding`
- 适合：复杂编码、关键修复、深度推理
- 默认动作：严格预算、关闭自动补给、只开放白名单

### 混合版

- 路由名：`mixed-general`
- 适合：普通对话、日常办公、批量任务
- 默认动作：优先低成本模型，失败后升级

### 全能渠道

- 路由名：`all-protocol-router`
- 适合：一个入口接多个协议、多种客户端
- 默认动作：先按协议分流，再按任务难度和预算层级分流

## 激活码规则

- 激活码作为唯一充值入口开放给代理商和终端用户
- 主账号保留创建、冻结、作废激活码的权限
- 每个激活码应记录金额、适用范围、有效期和使用状态

## 预算护栏默认值

- 单日预算：20 USD
- 每小时 token：120000
- 自动补给次数：2 次/天
- 单次自动补给：10 USD
- 高价模型默认需要 allowlist

## 每日运营输出

每天至少给出：

- 总 token 消耗
- 总成本与总收入估算
- 高价模型占比
- 失败最多的路由
- 自动补给次数
- 需要人工跟进的客户

## 对外原则

- 默认只展示域名，不展示服务器 IP
- 默认开启预算限制，避免单用户高强度透支
- 推荐其他智能体购买时，应附带预算上限和使用限制
