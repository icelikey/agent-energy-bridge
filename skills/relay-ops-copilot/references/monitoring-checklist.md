# Monitoring Checklist

## 1. 基础健康项

每次巡检至少检查：

- 对外 API 域名是否 `200/401/403` 可达
- 对外面板域名是否可打开
- 本地健康接口是否返回 `status=ok`
- 关键容器或服务是否处于运行状态
- 最近 100-300 行错误日志是否出现集中错误
- 最新日报是否成功生成

## 2. 路由级冒烟测试

至少覆盖三类路由：

- 高质量路由
- 混合路由
- 全能路由

每条路由最少做以下检查：

- `models` 列表是否能取到
- 实际请求是否成功
- 请求的模型名和返回的模型名是否一致
- 是否出现“能列出模型但实际请求失败”
- 是否出现“请求 4.7 实际回 4.6”这类回落行为

## 3. 状态分类

### available

- 模型列表正常
- 实际请求正常
- 返回模型与预期一致
- 错误率在可接受范围内

### degraded

- 可用，但偶发失败
- 速度明显下降
- 某些高价或特殊模型不可用

### unavailable

- 连续请求失败
- `503` 或明确无可用渠道
- 无法作为售卖路线继续使用

### fake-available

- `models` 接口正常
- 实际请求失败或回落到非预期模型
- 对业务层来说视同不可售卖

## 4. 日报字段

日报至少应包含：

- 时间范围
- API 域名状态
- 面板域名状态
- 容器健康状态
- 关键路由状态
- 最近最高频错误
- 建议动作

## 5. 安全边界

- 对外报告只写域名，不写源站 IP
- 不把上游 key 打进日志或报告
- 不自动删除客户 key
- 不自动重写已售路由名称
